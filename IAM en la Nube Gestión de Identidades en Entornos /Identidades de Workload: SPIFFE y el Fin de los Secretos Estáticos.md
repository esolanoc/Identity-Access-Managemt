Las identidades humanas no son las únicas que necesitan gestión en la nube. Los microservicios, contenedores, funciones serverless y aplicaciones automatizadas también necesitan autenticarse mutuamente y con servicios externos. Tradicionalmente, esto se lograba mediante credenciales estáticas: contraseñas codificadas, claves API hardcoded, certificados de larga duración. Cada una representa un riesgo de seguridad y un dolor operativo cuando debe rotarse.

## SPIFFE (Secure Production Identity Framework For Everyone):
Es un estándar diseñado para eliminar este problema. SPIRE, su implementación de producción, emite "SVIDs" (SPIFFE Verifiable Identity Documents) que son certificados X.509 o tokens JWT que identifican workloads sin intervención humana.
El proceso es elegante: SPIRE Agents desplegados en cada nodo realizan "node attestation" usando documentos de identidad del proveedor cloud (AWS Instance Identity Document, Azure Managed Service Identity, GCE Instance Identity Token, Kubernetes Service Account tokens). Una vez validado el nodo, realizan "workload attestation" para verificar propiedades del proceso solicitante: user ID, namespace de Kubernetes, imagen de contenedor. Solo entonces emiten SVIDs que el workload consume llamando al SPIFFE Workload API a través de un socket Unix.
SPIFFE Federation permite que workloads de diferentes dominios de confianza (trust domains) se autentiquen mutuamente, construyendo service mesh entre organizaciones sin compartir claves secretas.
